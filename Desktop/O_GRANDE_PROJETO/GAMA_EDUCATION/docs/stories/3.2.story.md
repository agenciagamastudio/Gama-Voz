## 📋 Description

Com os cursinhos reais funcionando (Story 3.1), o próximo passo natural é recompensar o aluno que completa 100% das lições. Esta story implementa emissão automática de certificado PDF quando o usuário conclui todas as lições de um cursinho.

O certificado deve:
- Ser emitido automaticamente na última chamada de `POST /complete` que fecha o cursinho
- Ser disponibilizado para download via endpoint autenticado
- Aparecer em uma página de perfil de certificados

**Contexto arquitetural real (descoberto em Story 3.1):** o backend usa armazenamento in-memory (Maps), não PostgreSQL/Neo4j. Certificados serão armazenados em um `Map<userId, Certificate[]>` da mesma forma que `userProgress`. Geração do PDF acontece on-demand com PDFKit.

## ✅ Acceptance Criteria

### AC1 — Detecção de Cursinho Completo

**Given** o usuário tem autenticação válida
**When** `POST /api/v1/courses/:slug/lessons/:lessonId/complete` é chamado e é a última lição pendente
**Then:**
- [x] Sistema calcula `completedCount / totalLessons` para o slug
- [x] Se ratio == 1.0 (100%): certificado é emitido automaticamente
- [x] Response inclui `{ ..., course_completed: true, certificate_id: "<uuid>" }` quando completo
- [x] Se ratio < 1.0: response inclui `{ ..., course_completed: false }` (sem certificate_id)
- [x] Certificado não é emitido novamente em chamadas subsequentes (idempotente)

### AC2 — Geração do PDF

**Given** um certificado foi emitido para o usuário
**When** `GET /api/v1/certificates/:certificateId/download` é chamado com token válido
**Then:**
- [x] Response tem `Content-Type: application/pdf`
- [x] Response tem `Content-Disposition: attachment; filename="certificado-{slug}-{name}.pdf"`
- [x] PDF contém: nome do aluno, nome do cursinho, data de conclusão, ID único do certificado
- [x] PDF é gerado com PDFKit (instalado: `npm install pdfkit @types/pdfkit`)
- [x] Response time < 500ms (geração simples, sem templates pesados)

### AC3 — Listagem de Certificados do Usuário

**Given** o usuário tem certificados emitidos
**When** `GET /api/v1/certificates` é chamado com token válido
**Then:**
- [x] Retorna array de certificados do usuário autenticado
- [x] Cada item tem: `{ id, course_slug, course_title, issued_at, download_url }`
- [x] `download_url` é `/api/v1/certificates/:id/download`
- [x] Se usuário não tem certificados: retorna `[]` (não 404)

### AC4 — Página de Certificados no Frontend

**Given** o usuário está autenticado e tem certificados
**When** acessa `/cursinhos/certificados`
**Then:**
- [x] Página lista os certificados com nome do cursinho e data
- [x] Botão "Baixar PDF" em cada certificado dispara download
- [x] Página vazia exibe mensagem motivacional ("Complete um cursinho para ganhar seu certificado!")
- [x] Link para a página aparece no dashboard `/cursinhos` (botão "🏆 Meus Certificados" no header)

### AC5 — Segurança

**Given** certificados contêm dados pessoais (nome do aluno)
**When** qualquer endpoint de certificado é acessado
**Then:**
- [x] Todos os endpoints exigem autenticação JWT válida (authMiddleware)
- [x] Usuário só acessa seus próprios certificados (validação por `userId`)
- [x] `GET /api/v1/certificates/:id/download` valida que o certificado pertence ao usuário autenticado
- [x] `404` retornado se certificado não existe OU pertence a outro usuário (não vazar existência)

## 🧪 Dev Notes

**Stack relevante:**
- PDFKit: `npm install pdfkit @types/pdfkit` — geração de PDF em Node.js, API de streams
- PDFKit básico gera PDF com `doc.text()`, `doc.font()`, etc.
- Stream do PDF pode ser enviado diretamente via `doc.pipe(res)`

**Pattern para geração de PDF (seguir):**
```typescript
import PDFDocument from 'pdfkit';

app.get('/api/v1/certificates/:id/download', authMiddleware, (req: any, res: Response) => {
  // 1. validar que cert existe e pertence ao userId
  const cert = findCertificate(req.params.id, req.userId);
  if (!cert) return res.status(404).json({ error: 'Certificate not found' });

  // 2. configurar response headers
  res.setHeader('Content-Type', 'application/pdf');
  res.setHeader('Content-Disposition', `attachment; filename="certificado-${cert.courseSlug}.pdf"`);

  // 3. gerar e streamar PDF
  const doc = new PDFDocument();
  doc.pipe(res);
  doc.fontSize(24).text('Certificado de Conclusão', { align: 'center' });
  // ... resto do conteúdo
  doc.end();
});
```

**Detecção de 100% no `POST /complete` (modificação mínima):**
```typescript
// Após adicionar a lição ao Set de progresso:
const courseData = courseBySlug.get(req.params.slug)!;
const totalLessons = courseData.modules.reduce((sum, m) => sum + m.lessons.length, 0);
const completedForCourse = Array.from(progress).filter(k => k.startsWith(`${req.params.slug}/`)).length;
const courseCompleted = completedForCourse === totalLessons;
// Se courseCompleted && !alreadyCertified → emitir certificado
```

**Padrão de dados in-memory (seguir o padrão de `userProgress`):**
```typescript
const certificates = new Map<string, Certificate>(); // certId → Certificate
const userCertificates = new Map<string, Set<string>>(); // userId → Set<certId>
```

**Endpoint de listagem — seguir padrão existente:**
```typescript
app.get('/api/v1/certificates', authMiddleware, (req: any, res: Response) => {
  const certIds = userCertificates.get(req.userId) || new Set<string>();
  const certs = Array.from(certIds).map(id => certificates.get(id)!).filter(Boolean);
  return res.json(certs.map(c => ({
    id: c.id,
    course_slug: c.courseSlug,
    course_title: c.courseTitle,
    issued_at: c.issuedAt,
    download_url: `/api/v1/certificates/${c.id}/download`,
  })));
});
```
